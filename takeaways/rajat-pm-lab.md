# My GitHub 101 Takeaway

## The Google Drive Framework

Just read a breakdown of GitHub and took step by step Github set up course using Claude Code. It mapped every concept to Google Drive, reframing how I think about development/product team workflows.

Branches are suggestion mode. Commits are save points. Pull requests are review cycles. Merging is publishing.

## The Playbook We Already Know

As PMs, we already run this playbook instinctively: we never edit the source of truth directly, checkpoint our work in between, get buy-in before shipping. Git just codifies it for all of us.

The teams that ship cleanly aren't the ones with the best tools but rather they're the ones with the best habits around protecting their main branch, whether that's a codebase, a PRD, or a product roadmap.

## Real Value for Product Leaders

In my opinion, the real value for product leaders from this course is understanding this workflow and immersing as a tourist in your engineering team's process and start being and not merely act as a collaborator. We can contribute to docs repos, review changes with a deeper context, and speak the same language in standup. Doesn't that help all of us?

Thank you so much and I cannot wait to contribute more in the weeks/months to come.
